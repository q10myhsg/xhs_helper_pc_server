## 系统要求

### 硬件要求
- **操作系统**：Windows 10/11、macOS 10.14+ 或 Linux
- **内存**：至少 4GB RAM（推荐 8GB+）
- **磁盘空间**：至少 2GB 可用空间
- **网络**：稳定的互联网连接

### 软件要求
- **Python**：3.7+（打包版不需要单独安装）
- **Android 设备**：Android 7.0+（用于养号功能）

## 快速开始

### 1. 下载程序

从 GitHub Releases 页面下载最新版本的程序：
- Windows: `xhs_helper_vx.x.x_windows_x64.zip`
- macOS: `xhs_helper_vx.x.x_macos_x64.zip` 或 `xhs_helper_vx.x.x_macos_arm64.zip`

### 2. 解压程序

将下载的压缩包解压到您喜欢的位置，例如：
- Windows: `C:\xhs_helper`
- macOS: `/Applications/xhs_helper`

### 3. 启动程序

双击运行程序文件：
- Windows: `xhs_helper.exe`
- macOS: `xhs_helper.app`

程序启动后会自动打开环境配置工具。

## 环境依赖配置

### 自动配置（推荐）

程序内置了环境配置工具，可以自动检测和安装所需的依赖项。

#### 步骤 1：检查环境

启动程序后，点击「检查环境」按钮，系统会自动检查以下依赖项：
- ✓ PYTHON：Python 环境
- ✗ ADB：Android 调试桥（用于连接 Android 设备）
- ✗ POPPLER：PDF 处理工具
- ✗ GIT：版本控制工具（可选）

#### 步骤 2：安装缺失依赖

1. 在缺失的依赖项列表中勾选需要安装的组件（推荐安装 ADB 和 Poppler）
2. 点击「安装选中组件」按钮
3. 等待安装完成

#### 步骤 3：验证安装

安装完成后，再次点击「检查环境」按钮，确认所有核心依赖项都显示为 ✓。

---

### 手动配置

如果自动安装失败，您也可以手动安装依赖项。

#### 安装 ADB

**Windows:**
1. 访问 https://developer.android.com/studio/releases/platform-tools
2. 下载 Windows 版本的 Platform Tools
3. 解压到某个目录，例如 `C:\adb`
4. 将该目录添加到系统 PATH 环境变量

**macOS:**
```bash
# 使用 Homebrew 安装
brew install --cask android-platform-tools
```

**Linux (Ubuntu/Debian):**
```bash
sudo apt-get install android-tools-adb
```

#### 安装 Poppler

**Windows:**
1. 访问 https://github.com/oschwartz10612/poppler-windows/releases
2. 下载最新版本的 Release 压缩包
3. 解压到某个目录
4. 将解压后的 `Library\bin` 目录添加到系统 PATH 环境变量

**macOS:**
```bash
# 使用 Homebrew 安装
brew install poppler
```

**Linux (Ubuntu/Debian):**
```bash
sudo apt-get install poppler-utils
```

## Android 设备配置

### 1. 启用开发者选项

1. 打开 Android 设备的「设置」
2. 进入「关于手机」
3. 连续点击「版本号」7 次，直到提示「您已处于开发者模式」

### 2. 开启 USB 调试

1. 返回「设置」主页面
2. 进入「系统」或「更多设置」
3. 找到并进入「开发者选项」
4. 启用「USB 调试」开关

### 3. 连接设备

1. 使用 USB 数据线将 Android 设备连接到电脑
2. 在设备上弹出的「允许 USB 调试」对话框中，点击「允许」或「确定」
3. （可选）勾选「始终允许使用这台计算机进行调试」

### 4. 验证连接

在程序的设备管理页面，点击「刷新设备列表」按钮，应该能看到您的设备。

## 使用说明

### 1. 启动 Web 服务

环境配置完成后，程序会自动启动 Web 服务。您也可以手动启动：
- 在程序界面点击「启动服务」按钮

服务启动后，会在浏览器中自动打开管理界面，默认地址为：`http://localhost:5002`

### 2. 设备管理

- **刷新设备列表**：点击按钮获取已连接的设备
- **选择设备**：在列表中选择要操作的设备
- **设置别名**：为设备设置易于识别的名称

### 3. 关键词配置

- **添加关键词**：在输入框中输入关键词，点击「添加」
- **批量导入**：在文本区域输入多个关键词（每行一个），点击「批量导入」
- **删除关键词**：点击关键词右侧的「删除」按钮

### 4. 启动养号

1. 在仪表板页面选择要养号的设备
2. 配置养号参数（时长、访问比例、互动概率等）
3. 点击「启动养号」按钮

### 5. 停止养号

- 在仪表板页面选择要停止养号的设备
- 点击「停止养号」按钮

## 常见问题

### Q: 程序无法启动怎么办？

**A:** 请确保：
1. 您的系统满足最低要求
2. 已正确解压程序文件
3. Windows 用户检查是否被杀毒软件拦截
4. macOS 用户在「系统偏好设置」→「安全性与隐私」中允许运行

### Q: 设备无法连接？

**A:** 请检查：
1. USB 线是否连接稳定
2. 设备是否已启用 USB 调试
3. 设备上是否允许了 USB 调试授权
4. 尝试更换 USB 线或 USB 接口
5. 运行 `adb devices` 命令查看设备状态

### Q: ADB 安装失败？

**A:** 如果自动安装失败，请尝试：
1. 手动安装 ADB（参考「手动配置」章节）
2. 检查网络连接
3. 尝试使用 VPN 或代理
4. 从其他镜像源下载

### Q: Poppler 安装失败？

**A:** 如果自动安装失败，请尝试：
1. 手动安装 Poppler（参考「手动配置」章节）
2. 如果不需要 PDF 处理功能，可以跳过安装
3. 检查网络连接

### Q: 养号过程中出现错误？

**A:** 请检查：
1. 设备是否已解锁
2. 小红书应用是否已安装并正常运行
3. 设备网络连接是否稳定
4. 查看程序日志了解具体错误信息

### Q: 如何更新程序？

**A:** 更新步骤：
1. 从 GitHub Releases 下载最新版本
2. 备份您的配置文件（如果需要）
3. 关闭正在运行的程序
4. 解压新版本覆盖旧版本
5. 重新启动程序

### Q: 程序支持多个设备同时养号吗？

**A:** 是的，程序支持同时管理多个 Android 设备。您可以：
1. 连接多个 Android 设备到电脑
2. 在设备列表中选择多个设备
3. 分别为每个设备配置养号参数
4. 同时启动多个设备的养号流程
